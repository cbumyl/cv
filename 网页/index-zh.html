QuickMart 即时零售新用户首单转化与留存诊断

围绕即时零售平台 3 月新用户增长质量波动，搭建从数据清洗、指标计算、异常诊断、原因下钻、AB 实验到看板汇报的完整分析链路。

  • 数据链路：基于公开资料调研校准的合成即时零售数据，使用 DuckDB + SQL 构建 10 张原始表、清洗层和分析集市，覆盖用户、行为、订单、商品、渠道、库存、AB 分组和客诉数据。
  • 指标诊断：构建新用户首单支付率、D1 留存、渠道贡献、漏斗转化、缺货率和实验 lift 指标体系；识别信息流渠道 A 为 D1 下滑最大负贡献渠道，加权贡献 -0.0167。
  • 漏斗定位：对信息流渠道 A 拆解浏览、加购、结算、支付链路，定位最大断点为 结算 -> 支付下降 16.80pp；首单用户 D1 留存 44.46%，明显高于未首单用户 21.56%。
  • 原因验证：交叉库存、城市、类目和客诉数据，发现爆款 SKU 缺货率由 8.04% 升至 21.73%，杭州、上海、深圳等重点城市出现缺货上升与转化下降同向。
  • 实验评估：设计“库存提示 + 替代推荐 + 优惠券提醒”组合方案并按 ITT 口径评估；实验组首单支付率从 4.05% 提升至 4.69%，相对提升 15.8%。
  • 交付沉淀：输出 6 页作品集看板、7 页汇报 PPT、指标口径字典、面试讲解稿和 GitHub 交付版；同时沉淀数据分析项目诊断 Skill 与合成数据设计 Skill。

Apple Intelligence 增长策略与发布路径设计

围绕 AI 手机竞争格局变化，完成资料交叉验证、趋势拆解、战略建议与商业测算，输出英文报告与商赛级 PPT。

个人复盘成长 App

定位为数据分析与商分研究效率工具，后续个人网页升级时会单独展开，用于展示 AI Agent、提示词调试和 Skill 自动化能力。

  • Problem：学习与项目信息分散，难以稳定复用，导致分析效率和复盘质量波动。
  • Approach：拆解“采集 - 结构化 - 检索 - 输出”流程，基于开源项目与个人需求设计知识管理、项目复盘和数据分析工作流。
  • Key Findings：高频低价值动作是主要效率损耗点，可通过 Agent 自动化、提示词模板和 Skill 封装降低重复劳动。
  • Recommendations：以多 Agent 协同替代重复流程，把更多时间集中在业务判断、数据验证、策略推演与汇报表达。
  • Metrics：通过内容增长与复盘产出验证效果：单条 AI 教学内容播放量 3 万+，单条点赞 + 收藏 3000+。
