ByteView 内容平台创作者增长与推荐分发效率诊断

围绕内容平台泛知识垂类推荐效率下滑，完成从数据建模、SQL 数据底座、指标体系、异常定位、创作者供给归因、推荐漏斗下钻到实验评估和看板交付的完整链路。

GitHub：byteview-portfolio

  • 数据建模与校验：基于公开资料与内容平台行业指标调研，使用内容平台公开业务数据表，自主构建用户、创作者、内容、审核、曝光、播放、互动、关注、负反馈、创作者激励及 AB 实验等 12 张原始业务表，覆盖 10 万用户、1 万创作者、11.2 万内容、208.97 万曝光事件、15.10 万播放事件及 730 名 AB 实验创作者，原始数据规模约 321 万行；通过固定随机种子保证数据可复现，设计并优化 Python 校验脚本，输出 Markdown 校验报告，完成主键、外键、日期范围、链路一致性、实验互斥等全量检查，最终质量检查结果为 33 项 PASS、4 项 WARN、0 项 FAIL。
  • 数据清洗与链路搭建：使用 DuckDB + SQL 建立 raw、clean、mart 三层数据链路，构建 30 张数据库表、全库约 642 万行；在 clean 层保留并标记 2056 条孤儿曝光、识别 116 条异常播放、映射 7 条异常反馈类型，保证脏数据可追溯且核心指标默认过滤；修正平台 DAU 跨垂类重复计数问题，将原 SUM 口径高估约 43.8% 的问题改为直接 COUNT DISTINCT 用户口径，并统一互动、播放、曝光等链路口径，导出 11 张聚合结果表支撑后续看板分析。
  • 指标体系与异常定位：搭建 CTR、3 秒留存率、完播率、互动率、负反馈率、曝光份额、投稿创作者占比、优质内容占比等 10+ 核心指标体系，按 Normal / Anomaly / Recovery 阶段进行异动对比，避免不等长窗口总量直接比较导致误判；通过垂类拆解定位异常集中在 泛知识垂类，其 CTR 从 0.0721 降至 0.0671（下降 0.49pp），完播率从 0.3308 降至 0.2204（下降 11.04pp），曝光份额从 24.9% 降至 20.3%（下降 4.6pp），而全平台 DAU 基本稳定，判断异常不是平台整体流量崩盘。
  • 创作者供给归因与推荐漏斗下钻：围绕“腰部创作者供给下降 -> 内容池收缩 -> 曝光迁移 -> 消费效率下降”构建归因链路，发现泛知识腰部创作者日均投稿从 287.8 降至 154.7（下降 46.2%），投稿创作者占比从 44.88% 降至 30.95%（下降 13.93pp），审核通过率下降 5.51pp；进一步拆解推荐页曝光结构，发现腰部曝光份额从 53.4% 降至 32.3%（下降 21.2pp），头部曝光份额从 37.3% 升至 55.7%（上升 18.4pp），说明推荐系统使用头部内容补位，CTR 被部分对冲但完播率显著下降，为内容质量与推荐匹配优化提供证据链。
  • AB 实验与交付沉淀：设计并分析面向泛知识腰部创作者的 EXP_001 激励实验，样本为 Control 365 人与 Treatment 365 人，Treatment 策略包含稳定冷启动流量包、垂类创作任务和数据反馈提醒；采用日人均口径评估实验效果，实验后 Treatment 组日均投稿率从 Control 的 0.3112 提升至 0.3633（提升 5.2pp），优质内容占比从 13.62% 提升至 18.18%（提升 4.6pp），完播率方向性提升 2.08pp，但 CTR 从 6.47% 降至 6.19%（下降 0.28pp），明确得出“供给侧激励有效，但点击效率需单独优化”的结论；最终输出 4 页 Tableau 风格看板、11 张聚合 CSV、指标口径字典、SQL 脚本及 GitHub 公开交付版，完成内容平台异常诊断项目的全链路沉淀。

QuickMart 即时零售新用户首单转化与留存诊断

围绕即时零售平台 3 月新用户增长质量波动，搭建从数据清洗、指标计算、异常诊断、原因下钻、AB 实验到看板汇报的完整分析链路。

  • 数据链路：基于公开资料调研校准的合成即时零售数据，使用 DuckDB + SQL 构建 10 张原始表、清洗层和分析集市，覆盖用户、行为、订单、商品、渠道、库存、AB 分组和客诉数据。
  • 指标诊断：构建新用户首单支付率、D1 留存、渠道贡献、漏斗转化、缺货率和实验 lift 指标体系；识别信息流渠道 A 为 D1 下滑最大负贡献渠道，加权贡献 -0.0167。
  • 漏斗定位：对信息流渠道 A 拆解浏览、加购、结算、支付链路，定位最大断点为 结算 -> 支付下降 16.80pp；首单用户 D1 留存 44.46%，明显高于未首单用户 21.56%。
  • 原因验证：交叉库存、城市、类目和客诉数据，发现爆款 SKU 缺货率由 8.04% 升至 21.73%，杭州、上海、深圳等重点城市出现缺货上升与转化下降同向。
  • 实验评估：设计“库存提示 + 替代推荐 + 优惠券提醒”组合方案并按 ITT 口径评估；实验组首单支付率从 4.05% 提升至 4.69%，相对提升 15.8%。
  • 交付沉淀：输出 6 页作品集看板、7 页汇报 PPT、指标口径字典、面试讲解稿和 GitHub 交付版；同时沉淀数据分析项目诊断 Skill 与合成数据设计 Skill。
